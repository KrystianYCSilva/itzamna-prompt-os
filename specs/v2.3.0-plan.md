# v2.3.0 — Advanced Features & Ecosystem

> **Release plan for PromptOS v2.3.0**
> **Status:** Planning | **Estimate:** 5-7 days | **Created:** 2026-02-03 (Session 25)
> **Depends on:** v2.2.0 complete (SPEC-003 + Go baseline)

---

## Motivation

v2.2.0 proved two things end-to-end:

1. **SPEC-003 protocols work in production** — Go baseline scored 100/100 on its first application.
2. **JIT sub-files scale** — WEB-RESEARCH main file stayed at 1,393 tokens while 4 sub-files carry depth.

The same pattern applies to language skills. Baselines are solid (6/6, 99.20 avg); what's missing is the **ecosystem layer** that developers actually reach for daily, and coverage of three high-demand languages that have no baseline yet.

## Phase 2 — New Baseline Skills

**Estimate:** 2 days | **Deliverables:** 3 SKILL.md files | **Pattern:** same as existing 6 baselines

Apply SPEC-003 research protocols (WEB-RESEARCH.md v2.1) and SPEC-010 template for each.

### Rust Baseline

| Section | Content |
|---------|---------|
| Core concepts | Ownership, borrowing, lifetimes, move semantics |
| Type system | Enums (Option/Result), traits, generics, pattern matching |
| Concurrency | threads, Arc/Mutex, async/await (tokio overview) |
| Memory model | Stack vs heap, Box, Rc, no GC — zero-cost guarantees |
| Error handling | `?` operator, custom Error types, anyhow/thiserror |
| Comparison | vs C++ (memory safety), vs Go (performance), vs Java (ownership) |

**Sources (T1):** doc.rust-lang.org, The Rust Book

### TypeScript Baseline

| Section | Content |
|---------|---------|
| Core concepts | Static typing over JS: interfaces, type aliases, enums |
| Advanced types | Generics, mapped types, conditional types, utility types |
| Classes & modules | Decorators, abstract classes, namespaces vs modules |
| Async | Promises, async/await, error handling patterns |
| Compilation | tsconfig.json, strict mode, target options |
| Comparison | vs JavaScript (type safety), vs Java (structural typing), vs Rust (null safety) |

**Sources (T1):** typescriptlang.org, handbook

### Ruby Baseline

| Section | Content |
|---------|---------|
| Core concepts | Everything is an object, blocks, Procs, lambdas |
| OOP | Classes, modules/mixins, method_missing, open classes |
| Iteration | Enumerable, each/map/select/reduce, lazy enumerators |
| Metaprogramming | define_method, class_eval, DSL patterns |
| Error handling | begin/rescue/ensure, custom exceptions |
| Comparison | vs Python (philosophy), vs JavaScript (dynamic typing), vs Go (simplicity) |

**Sources (T1):** ruby-doc.org, The Ruby Way

## Phase 3 — Version-Specific Advanced Skills

**Estimate:** 1-2 days | **Deliverables:** 3 SKILL.md files | **Pattern:** same as `java-17`, `kotlin-2xx`

These live as sub-folders of their baseline (e.g., `go/go-118/SKILL.md`).

### Go 1.18+ (`go/go-118/SKILL.md`)

- Generics: type parameters, constraints, `any` / `comparable`
- Fuzzing: `testing.F`, corpus management
- Workspace mode: `go.work`, multi-module repos
- Comparison to pre-1.18 patterns

### Python 3.10+ (`python/python-310/SKILL.md`)

- Structural pattern matching: `match`/`case`, guards, class patterns
- `ParamSpec` and `TypeGuard` (typing)
- `tomllib` stdlib addition
- Performance: specializing adaptive interpreter (3.11+)

### JavaScript ES2023+ (`javascript/es2023/SKILL.md`)

- Array methods: `findLast`, `findLastIndex`, `toSorted`, `toReversed`, `with`
- `Object.hasOwn`, `Error.cause`
- Top-level `await` in modules
- Comparison to previous ES versions

### Acceptance Criteria (Phase 3)

- [ ] Each file ≤ 1,400 tokens
- [ ] Self-Critique ≥ 95
- [ ] Human Gate approved
- [ ] Referenced from parent baseline SKILL.md
- [ ] INDEX.md updated (advanced skills grow by 3)

## Protocol Checklist (per deliverable)

Follow the mandatory 6-step sequence from AGENTS.md:

1. AUTO-INCREMENT — verify no duplicate in INDEX.md
2. GENERATE — use SKILL.template.md, apply SPEC-003 citations
3. SELF-CRITIQUE — 4 dimensions, score ≥ 95
4. HUMAN-GATE — show preview, wait for `approve`
5. COMMIT — write file, update INDEX.md, conventional commit
6. MEMORY-MANAGEMENT — update MEMORY.md + opencode-memory.md
